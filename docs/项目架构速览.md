# ShadowLink 项目架构速览

本文面向后续开发者，帮助快速理解仓库结构、服务边界、主要入口与常见改动位置。

## 1. 项目定位

ShadowLink 是一个本地优先的 AI 工作台，当前主线是三端架构：

- `shadowlink-web`：React + Vite 前端，默认端口 `3000`。
- `shadowlink-ai`：FastAPI AI 服务，默认端口 `8000`。
- `shadowlink-server`：Spring Boot 网关/业务服务，默认端口 `8080`。
- `shadowlink-electron`：Electron 桌面壳，当前偏预留/桌面化集成。

仓库根目录仍保留早期 PyQt/本地脚本形态的文件，例如 `main.py`、`llm/`、`rag/`、`storage/`、`tools/`。新功能开发优先看 `shadowlink-web`、`shadowlink-ai`、`shadowlink-server`。

## 2. 顶层目录职责

| 路径 | 职责 |
| --- | --- |
| `shadowlink-web/` | Web 前端，页面、组件、状态管理、前端 API/SSE/WebSocket 客户端。 |
| `shadowlink-ai/` | Python AI 服务，Agent、RAG、MCP、文件处理、面试学习、代码库分析等能力。 |
| `shadowlink-server/` | Java 后端，多模块 Maven 项目，承载认证、会话、网关代理、业务 API、WebSocket。 |
| `shadowlink-electron/` | Electron 主进程/桌面打包配置，后续将 Web 能力桌面化。 |
| `docs/` | 设计文档、执行方案、工作记录。本文也放在这里。 |
| `deploy/` | Docker Compose、Nginx、MySQL、Redis、Prometheus 等部署配置。 |
| `proto/` | gRPC Protobuf 定义，生成代码输出到 AI 服务等位置。 |
| `assets/` | 图标、图片等共享静态资源。 |
| `data/`、`rag_indexes/` | 本地数据、上传文件、FAISS/RAG 索引等运行期数据。 |
| `config/` | 配置模板与旧架构配置说明。 |
| `scripts/` | 辅助脚本。 |

## 3. 整体请求链路

开发模式下推荐理解为：

```text
浏览器
  -> shadowlink-web:3000 (Vite)
    -> /api、/api/ai、/ws 代理到 shadowlink-server:8080
    -> /v1 直接代理到 shadowlink-ai:8000

shadowlink-server:8080
  -> 业务接口、会话、认证、WebSocket
  -> /api/ai/* 代理/桥接到 shadowlink-ai:8000

shadowlink-ai:8000
  -> /v1/agent、/v1/rag、/v1/interview、/v1/file 等 AI 能力接口
```

关键代理配置在 `shadowlink-web/vite.config.ts`：

- `/api/ai` -> Java 服务 `http://localhost:8080`。
- `/api` -> Java 服务 `http://localhost:8080`。
- `/ws` -> Java WebSocket `http://localhost:8080`。
- `/v1` -> Python AI 服务 `http://localhost:8000`，用于尚未通过 Java 映射的 AI 路由。

## 4. Web 前端结构

入口文件：

- `shadowlink-web/src/main.tsx`：React 挂载入口。
- `shadowlink-web/src/App.tsx`：路由、React Query、全局主题、AmbientProvider。
- `shadowlink-web/vite.config.ts`：路径别名 `@ -> src`、开发代理、构建分包。

主要目录：

| 路径 | 职责 |
| --- | --- |
| `src/pages/` | 页面级组件，当前路由包括聊天、面试学习、阅读、知识库、设置。 |
| `src/components/layout/` | 应用框架、侧边栏/布局。 |
| `src/components/chat/` | 聊天 UI。 |
| `src/components/agent/` | Agent 模式相关 UI。 |
| `src/components/interview/` | 面试学习、阅读工作台、Job Space、简历/JD/Profile、Expert Mode 等组件。 |
| `src/components/ambient/` | Ambient 工作模式 Provider/组件。 |
| `src/services/` | 前端请求封装，包括 REST、SSE、WebSocket、interview、codebase、session、integrations。 |
| `src/stores/` | Zustand 状态，包括 chat、agent、ambient、interview、settings。 |
| `src/hooks/` | 复用 Hook，例如 Agent、Ambient、WebSocket、External Runs。 |
| `src/types/` | TypeScript 类型定义。 |
| `src/theme/` | 主题配置。 |

路由在 `shadowlink-web/src/App.tsx`：

| URL | 页面 |
| --- | --- |
| `/`、`/chat` | ChatPage |
| `/interview` | InterviewLearningPage |
| `/reading` | ReadingPage |
| `/knowledge` | KnowledgePage |
| `/settings` | SettingsPage |

前端 API 基础封装在 `shadowlink-web/src/services/api.ts`，默认以 `/api` 为基础路径，返回体期望匹配 `ApiResult<T>`。

## 5. Python AI 服务结构

入口文件：

- `shadowlink-ai/app/main.py`：FastAPI app factory、middleware、exception handler、router 注册。
- `shadowlink-ai/app/config.py`：Pydantic Settings，读取 `.env` 与 `SHADOWLINK_*` 环境变量。
- `shadowlink-ai/pyproject.toml`：依赖、ruff、mypy、pytest、coverage 配置。

核心目录：

| 路径 | 职责 |
| --- | --- |
| `app/api/v1/` | REST API 路由层。 |
| `app/api/grpc/` | gRPC 服务实现与生成代码使用位置。 |
| `app/agent/` | Agent 编排，含 ReAct、Plan-and-Execute、多 Agent、Memory。 |
| `app/rag/` | RAG Pipeline，含 chunking、embedding、index、retrieval、reranking。 |
| `app/llm/` | 多 Provider LLM Client，OpenAI/DeepSeek/Ollama/Anthropic 等适配。 |
| `app/file_processing/` | 文件解析与抽取，PDF/DOCX/XLSX/PPTX/Markdown/代码/图片等。 |
| `app/mcp/` | MCP 协议与工具互操作。 |
| `app/tools/` | 工具执行层。 |
| `app/interview/` | 面试学习、题目生成、回答审阅、阅读工作台相关服务。 |
| `app/integrations/` | Codex、Web Research 等外部集成预留/适配。 |
| `app/codebase/` | 代码库分析/技术档案生成。 |
| `app/core/` | 日志、中间件、生命周期、异常等基础设施。 |
| `app/models/` | Pydantic/领域模型。 |
| `tests/` | Python 测试。 |

已注册的 REST 路由前缀在 `shadowlink-ai/app/main.py`：

| 前缀 | 路由文件 | 职责 |
| --- | --- | --- |
| `/health`、`/ready` | `health_router.py` | 健康检查。 |
| `/v1/agent` | `agent_router.py` | Agent Chat、流式输出、取消会话。 |
| `/v1/rag` | `rag_router.py` | RAG 检索/索引相关接口。 |
| `/v1/mcp` | `mcp_router.py` | MCP 工具列表、调用、插件。 |
| `/v1/file` | `file_router.py` | 文件处理、上传、入库。 |
| `/v1/integrations` | `integrations_router.py` | 外部集成状态，例如 Codex。 |
| `/v1/interview` | `interview_router.py` | 面试学习、Job Space、技能、项目文档、阅读工作台。 |
| `/v1/codebase` | `codebase_router.py` | 代码库 Profile 生成与查询。 |
| `/v1/settings` | `settings_router.py` | AI 服务设置。 |
| `/v1/system` | `system_router.py` | 系统信息/诊断类接口。 |

常见 AI 改动入口：

- 改 Agent 默认 Prompt：`shadowlink-ai/app/agent/engine.py`。
- 改 ReAct 执行逻辑：`shadowlink-ai/app/agent/react/`。
- 改 Plan & Execute：`shadowlink-ai/app/agent/plan_execute/`。
- 改多 Agent：`shadowlink-ai/app/agent/multi_agent/`。
- 改 LLM Provider：`shadowlink-ai/app/llm/providers/` 和 `shadowlink-ai/app/llm/client.py`。

- 改文件上传/RAG 入库：`shadowlink-ai/app/api/v1/file_router.py`、`shadowlink-ai/app/file_processing/`、`shadowlink-ai/app/rag/`。

## 6. Java 服务结构

入口文件：

- `shadowlink-server/pom.xml`：父 Maven 项目与模块聚合。
- `shadowlink-server/shadowlink-starter/src/main/java/com/shadowlink/ShadowLinkApplication.java`：Spring Boot 启动入口。
- `shadowlink-server/shadowlink-starter/src/main/resources/application.yml`：通用配置。
- `shadowlink-server/shadowlink-starter/src/main/resources/application-dev.yml`：开发环境，默认 H2 内存库。
- `shadowlink-server/shadowlink-starter/src/main/resources/application-prod.yml`：生产环境配置。

模块职责：

| 模块 | 职责 |
| --- | --- |
| `shadowlink-starter` | 应用启动模块，聚合其他模块并提供配置。 |
| `shadowlink-common` | 通用返回体、异常、配置、实体基类、工具类。 |
| `shadowlink-gateway` | 网关/代理层，CORS、限流、请求日志、AI Chat/Settings 代理。 |
| `shadowlink-auth` | 认证、JWT、登录/刷新/登出接口。 |
| `shadowlink-session` | 会话与消息管理。 |
| `shadowlink-business` | 业务能力，例如工作模式 `/api/modes`。 |
| `shadowlink-websocket` | WebSocket 配置与聊天消息处理。 |
| `shadowlink-ai-bridge` | Java 到 Python AI 服务的桥接客户端/DTO。 |

主要 Java API：

| 前缀 | 位置 | 职责 |
| --- | --- | --- |
| `/health` | `shadowlink-gateway/.../HealthController.java` | Java 服务健康检查。 |
| `/api/ai` | `shadowlink-gateway/.../ChatProxyController.java` | AI Chat/流式 Agent 代理到 Python。 |
| `/api/settings` | `shadowlink-gateway/.../SettingsProxyController.java` | 设置相关代理。 |
| `/api/auth` | `shadowlink-auth/.../AuthController.java` | 登录、刷新、登出。 |
| `/api/sessions` | `shadowlink-session/.../SessionController.java` | 会话 CRUD、消息查询。 |
| `/api/modes` | `shadowlink-business/.../WorkModeController.java` | 工作模式管理。 |
| `/ws` | `shadowlink-websocket/...` | WebSocket 通道。 |

## 7. Electron 桌面壳

`shadowlink-electron/` 当前是桌面化预留：

- `package.json` 中 `main` 指向 `dist/main/index.js`。
- `src/` 放 Electron 主进程代码。
- `electron-builder` 负责 Windows/macOS/Linux 打包。
- 资源来自 `../assets/icons`。

如果只是开发 Web/AI/Java 主线功能，通常不用改 Electron。

## 8. 运行与构建

根目录 `Makefile` 提供统一命令，但其中 Python 路径目前写死为本机 Conda 路径 `D:/software/study/Ana/envs/shadowlink/python.exe`。如果环境不同，需要先调整 Makefile 或手动执行对应命令。

常用命令：

```bash
# Python AI
cd shadowlink-ai
pip install -e ".[dev]"
uvicorn app.main:app --reload --port 8000

# Java Server
cd shadowlink-server
mvnw.cmd spring-boot:run

# Web
cd shadowlink-web
npm install
npm run dev
```

根目录 Makefile 目标：

```bash
make dev-ai
make dev-java
make dev-web
make build
make test
make lint
make docker-dev
```

Windows 用户也可以查看根目录 `start.bat`，它会尝试同时拉起 AI、Java、Web。

## 10. 常见开发任务导航

### 新增一个前端页面

1. 在 `shadowlink-web/src/pages/` 新建页面组件。
2. 在 `shadowlink-web/src/App.tsx` 添加路由。
3. 如需入口菜单，修改 `shadowlink-web/src/components/layout/`。
4. 如需请求后端，在 `shadowlink-web/src/services/` 添加服务封装，并在 `src/types/` 补类型。

### 新增一个 Python AI 接口

1. 在 `shadowlink-ai/app/api/v1/` 新增或修改 router。
2. 如是新 router，在 `shadowlink-ai/app/main.py` 的 `_register_routers()` 注册。
3. 业务逻辑放在对应领域目录，例如 `app/interview/`、`app/rag/`、`app/agent/`。
4. 前端若直接访问，走 `/v1/...`；若需要统一 Java 网关，再补 Java 代理。

### 新增一个 Java 业务接口

1. 选择模块：通用能力放 `common`，业务接口放 `business/session/auth/gateway` 等对应模块。
2. Controller 使用 `/api/...` 前缀，返回统一 `Result<T>`。
3. Service/DTO/Mapper 按现有模块风格补齐。
4. 前端通过 `shadowlink-web/src/services/api.ts` 的 `/api` 基础路径访问。

### 修改 Agent/LLM 行为

- Agent 策略入口：`shadowlink-ai/app/agent/engine.py`。
- ReAct：`shadowlink-ai/app/agent/react/`。
- Plan & Execute：`shadowlink-ai/app/agent/plan_execute/`。
- 多 Agent：`shadowlink-ai/app/agent/multi_agent/`。
- LLM Provider：`shadowlink-ai/app/llm/`。
- 前端 Agent UI/状态：`shadowlink-web/src/components/agent/`、`shadowlink-web/src/stores/agent-store.ts`、`shadowlink-web/src/hooks/useAgent.ts`。

### 修改面试学习/阅读工作台

- 前端页面/组件：`shadowlink-web/src/components/interview/`、`shadowlink-web/src/pages/`。
- 前端服务：`shadowlink-web/src/services/interview.ts`。
- 前端状态：`shadowlink-web/src/stores/interview-store.ts`。
- 后端 API：`shadowlink-ai/app/api/v1/interview_router.py`。
- 后端服务：`shadowlink-ai/app/interview/`。
- 设计文档：`docs/面试学习模块设计/`。

### 修改代码库分析/Codex 专家模式

- 前端服务/类型：`shadowlink-web/src/services/codebase.ts`、`shadowlink-web/src/types/codebase.ts`。
- Python API：`shadowlink-ai/app/api/v1/codebase_router.py`。
- Python 服务：`shadowlink-ai/app/codebase/`。
- Codex 集成：`shadowlink-ai/app/integrations/codex/`。

## 11. 注意事项

- 仓库里同时存在旧根目录 Python 桌面代码和新三端架构；新需求优先落在 `shadowlink-*` 子项目。
- `shadowlink-web/node_modules`、`shadowlink-web/dist`、`shadowlink-electron/node_modules` 等是生成物/依赖目录，不应手动改业务逻辑。
- 运行期数据在 `data/`、`shadowlink-ai/data/`、`rag_indexes/`，提交前注意不要误提交大体积索引或隐私数据。
- AI 服务配置大量依赖 `.env`/环境变量，命名多为 `SHADOWLINK_*`、`SHADOWLINK_LLM_*`、`SHADOWLINK_RAG_*` 等。
- 如果要改接口契约，优先同步修改：后端 DTO/模型、前端 `src/types/`、前端 `src/services/`、相关文档。
