# 面试学习模块执行方案

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在现有 ShadowLink 工程框架上，逐步扩充“分岗面试学习 + 阅读理解 + Codex 专家模式 + 网页能力预留”的完整模块。

**Architecture:** 保持现有 `shadowlink-ai`、`shadowlink-web`、`shadowlink-server` 分层，不推倒重来。优先在 Python AI 服务内完成业务闭环，前端通过 REST、短轮询、SSE 接入；Java 服务暂不作为强依赖，只保留未来网关 / 用户体系接入空间。

**Tech Stack:** FastAPI、SSE、React、TypeScript、Zustand、SQLite / JSON 本地持久化过渡层、LangChain / LangGraph 预留、Codex CLI、Chroma / 现有 RAG 模块。


## 1. 工程架构原则

- 不新建第二套后端服务。

## 2. 阶段路线图

### Phase 1：分岗 Space 与资料管理

目标：先把“不同岗位分别上传简历和 JD”的业务边界建起来。

交付物：

- Job Space 增删查改。
- 每个 Space 独立维护 Resume、JD、目标岗位、公司、备注。
- 前端能切换 Space，不同 Space 显示不同主题提示。
- 后端所有接口开始显式携带 `space_id`。

验收标准：

- 创建 `AI 工程师` 和 `PM` 两个 Space 后，二者 Resume / JD 不串线。
- 刷新页面后 Space 和 Profile 仍能恢复。
- 没有选择 Space 时，Agent 入口提示先创建或选择 Space。

### Phase 2：多 Session 隔离与短轮询状态

目标：让每个 Space 下可以有多个学习 / 面试 Session，且任务状态可短轮询。

交付物：

- Session 绑定 `space_id`。
- Task 状态模型：`queued`、`running`、`waiting_user_review`、`completed`、`failed`。
- 短轮询接口：查询 Session 状态、索引状态、外部 Agent 状态。
- 前端在任务运行时短轮询状态，不依赖 WebSocket。

验收标准：

- 同一个问题在不同 Space 下创建不同 Session。
- 前端每 1.5 到 3 秒刷新任务状态，任务结束后停止轮询。
- SSE 断开不影响后续通过状态接口看到任务完成 / 失败。

### Phase 3：面试 Agent 审阅闭环

目标：实现“原回答 vs 导师建议”的 Reviewer-in-the-Loop 产品闭环。

交付物：

- 面试回答提交接口。
- 审阅结果模型：`original_answer`、`suggested_answer`、`critique`、`sources`、`status`。
- SSE 输出 token / thought / tool / review / done。
- 前端展示对比视图，支持“采纳并更新”“保留原样”“重新生成”。

验收标准：

- Agent 不会自动覆盖用户原始回答。
- 用户点击采纳后才更新最终答案。
- 用户点击保留后保留原文，并记录导师建议。
- 用户输入二次指令后创建新的审阅版本。

### Phase 4：Codex CLI 专家模式

目标：在复杂项目场景下，调用本机 Codex CLI 做只读代码库深度分析。

交付物：

- Codex 安装检测接口。
- Codex 首次使用引导。
- `CodexExpertAdapter` 后端封装。
- `external_agent_runs` 运行记录。
- Codex JSONL 输出转 SSE。
- 前端 Expert Mode 面板。

验收标准：

- Windows 下优先检测并调用 `codex.cmd`。
- 未安装时前端显示安装步骤：Node.js、`npm install -g @openai/codex`、`codex login`。
- 已安装时可以选择一个本地项目路径，只读分析代码。
- 默认命令包含 `--sandbox read-only`，不修改文件。
- Codex 输出能进入现有 AgentPanel / ReviewPanel。

### Phase 5：网页搜索能力预留

目标：当前不购买外部 API，但工程上保留未来模型联网搜索能力。

交付物：

- `WebResearchProvider` 抽象。
- `DisabledWebResearchProvider` 默认实现。
- `sources` 数据模型。
- 前端显示“网页搜索未启用，可手动粘贴 URL / 正文”。
- 用户手动添加网页材料后，可归档到当前 Space。

验收标准：

- 未配置搜索 Provider 时，Agent 不会假装联网搜索。
- 前端能明确提示“当前未启用网页搜索”。
- 用户粘贴网页正文后，可以作为当前 Space 材料参与回答。
- 后续接 OpenAI / Claude / Gemini / DeepSeek 搜索时，不需要改业务层调用方式。

### Phase 6：阅读理解工作台

目标：实现第二赛道：极简阅读、点击句子、语义拆解、进度存根。

交付物：

- 阅读 Space 类型。
- 文章上传 / 粘贴。
- 句子切分与点击解释。
- 阅读进度、难句、标记记录。
- 前端纯净阅读模式。

验收标准：

- 阅读页面不出现复杂面试审阅元素。
- 点击句子后只解释当前句子，不污染全文。
- 阅读进度刷新后仍保留。

在 `shadowlink-ai/app/interview/repository.py` 实现最小持久化：

```python
from __future__ import annotations

import json
import uuid
from pathlib import Path

from app.interview.models import JobSpace, SpaceProfile, utc_now


class InterviewRepository:
    def __init__(self, data_dir: str | Path):
        self.data_dir = Path(data_dir)
        self.data_dir.mkdir(parents=True, exist_ok=True)
        self.spaces_file = self.data_dir / "spaces.json"
        self.profiles_file = self.data_dir / "profiles.json"

    def _load_list(self, path: Path) -> list[dict]:
        if not path.exists():
            return []
        return json.loads(path.read_text(encoding="utf-8"))

    def _save_list(self, path: Path, rows: list[dict]) -> None:
        path.write_text(json.dumps(rows, ensure_ascii=False, indent=2, default=str), encoding="utf-8")

    def create_space(self, name: str, type: str, theme: str) -> JobSpace:
        space = JobSpace(space_id=f"space-{uuid.uuid4().hex[:12]}", name=name, type=type, theme=theme)
        rows = self._load_list(self.spaces_file)
        rows.append(space.model_dump(mode="json"))
        self._save_list(self.spaces_file, rows)

        profiles = self._load_list(self.profiles_file)
        profiles.append(SpaceProfile(space_id=space.space_id).model_dump(mode="json"))
        self._save_list(self.profiles_file, profiles)
        return space

    def list_spaces(self) -> list[JobSpace]:
        return [JobSpace(**row) for row in self._load_list(self.spaces_file)]

    def get_profile(self, space_id: str) -> SpaceProfile:
        for row in self._load_list(self.profiles_file):
            if row["space_id"] == space_id:
                return SpaceProfile(**row)
        return SpaceProfile(space_id=space_id)

    def update_profile(self, profile: SpaceProfile) -> SpaceProfile:
        profile.updated_at = utc_now()
        rows = self._load_list(self.profiles_file)
        rows = [row for row in rows if row["space_id"] != profile.space_id]
        rows.append(profile.model_dump(mode="json"))
        self._save_list(self.profiles_file, rows)
        return profile
```

在 `shadowlink-ai/app/api/v1/interview_router.py` 增加：

```python
from __future__ import annotations

from pathlib import Path

from fastapi import APIRouter, HTTPException

from app.config import settings
from app.interview.models import SpaceProfile
from app.interview.repository import InterviewRepository
from app.interview.schemas import CreateSpaceRequest, SpaceDetail, UpdateProfileRequest
from app.models.common import Result

router = APIRouter(prefix="/interview", tags=["interview"])


def get_repo() -> InterviewRepository:
    return InterviewRepository(Path(settings.data_dir) / "interview")


@router.get("/spaces")
async def list_spaces() -> Result[list[SpaceDetail]]:
    repo = get_repo()
    details = [SpaceDetail(space=space, profile=repo.get_profile(space.space_id)) for space in repo.list_spaces()]
    return Result.ok(data=details)


@router.post("/spaces")
async def create_space(request: CreateSpaceRequest) -> Result[SpaceDetail]:
    repo = get_repo()
    space = repo.create_space(request.name, request.type.value, request.theme)
    return Result.ok(data=SpaceDetail(space=space, profile=repo.get_profile(space.space_id)))


@router.put("/spaces/{space_id}/profile")
async def update_profile(space_id: str, request: UpdateProfileRequest) -> Result[SpaceProfile]:
    repo = get_repo()
    if not any(space.space_id == space_id for space in repo.list_spaces()):
        raise HTTPException(status_code=404, detail="Space not found")
    profile = SpaceProfile(space_id=space_id, **request.model_dump())
    return Result.ok(data=repo.update_profile(profile))
```

```python
from __future__ import annotations

import platform
import shutil
import subprocess

from app.integrations.codex.schemas import CodexStatus


def detect_codex() -> CodexStatus:
    candidates = ["codex.cmd", "codex"] if platform.system().lower() == "windows" else ["codex"]
    for candidate in candidates:
        command = shutil.which(candidate)
        if not command:
            continue
        try:
            result = subprocess.run([command, "--version"], capture_output=True, text=True, timeout=5)
            version = (result.stdout or result.stderr).strip().splitlines()[0] if (result.stdout or result.stderr) else "unknown"
            return CodexStatus(installed=True, command=command, version=version)
        except Exception as exc:
            return CodexStatus(installed=True, command=command, message=str(exc))
    return CodexStatus(
        installed=False,
        message="Codex CLI not found. Install Node.js, run `npm install -g @openai/codex`, then run `codex login`.",
    )
```

```python
from __future__ import annotations

import asyncio
from collections.abc import AsyncIterator

from app.integrations.codex.detector import detect_codex


class CodexExpertAdapter:
    async def run_readonly(self, repo_path: str, prompt: str) -> AsyncIterator[dict]:
        status = detect_codex()
        if not status.installed or not status.command:
            yield {"event": "error", "data": {"message": status.message}}
            return

        process = await asyncio.create_subprocess_exec(
            status.command,
            "exec",
            "--cd",
            repo_path,
            "--sandbox",
            "read-only",
            "--ask-for-approval",
            "never",
            "--json",
            prompt,
            stdout=asyncio.subprocess.PIPE,
            stderr=asyncio.subprocess.PIPE,
        )

        assert process.stdout is not None
        async for line in process.stdout:
            yield {"event": "codex_event", "data": {"raw": line.decode("utf-8", errors="ignore").strip()}}

        return_code = await process.wait()
        yield {"event": "codex_done", "data": {"return_code": return_code}}
```

```python
from __future__ import annotations

import re


def split_reading_sentences(text: str) -> list[str]:
    normalized = re.sub(r"\s+", " ", text.strip())
    if not normalized:
        return []
    parts = re.split(r"(?<=[。！？!?\.])\s*", normalized)
    return [part.strip() for part in parts if part.strip()]
```

## 5. 风险控制

### 5.1 Codex CLI 风险

- 默认只读：`--sandbox read-only`。
- 默认不交互：`--ask-for-approval never`。
- 禁止危险参数：`--dangerously-bypass-approvals-and-sandbox`。
- Windows 调用 `codex.cmd`，不调用容易被执行策略拦截的 `codex.ps1`。
- repo 路径必须来自用户显式选择，并做 allowlist 校验。

### 5.2 Session 串线风险

- 所有 Interview 数据必须带 `space_id`。
- 所有 Agent Task 必须带 `session_id`。
- 后端写入前校验 `session.space_id == request.space_id`。
- 前端切换 Space 时清空当前审阅状态，重新加载该 Space 下 Session。

### 5.3 网页搜索幻觉风险

- 未启用 WebResearchProvider 时，Prompt 明确写入“当前不可联网搜索”。
- UI 明确展示“网页搜索未启用”。
- 没有真实 source 时，不展示来源卡片。
- 用户手动粘贴资料时，source provider 标为 `manual`。

### 5.4 工程膨胀风险

- 先做 JSON Repository，不急着一次性引入完整数据库迁移。
- Codex 专家模式不进入默认 Agent 链路。
- 阅读工作台不复用面试审阅复杂 UI。
- Java 服务暂不参与核心闭环，避免跨服务调试成本过高。
