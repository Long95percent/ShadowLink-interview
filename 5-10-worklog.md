# 5-10 Worklog

## 面试模拟与岗位管理

- 重构了模拟面试页的岗位状态模型，将右侧岗位设置面板拆成 `view / create / edit` 三态。
- 限制新岗位只能从左侧 `我的岗位 -> 新建` 入口创建，右侧默认不再具备全局创建能力。
- 右侧岗位设置现在绑定当前选中的岗位；切换岗位时展示对应岗位信息，编辑时只影响当前岗位。
- 增加岗位删除、编辑和保存后的状态同步，避免创建/编辑/保存逻辑混乱。
- 修复新建岗位前不能上传简历的问题：新增草稿简历解析接口，前端上传简历不再依赖已有 `space_id`。

## 模拟面试 UI 与收藏题仓库

- 调整模拟面试入口与全屏练习体验，题目卡片居中展示，减少重复入口和杂乱按钮。
- 收藏题目仓库只保留一个明确入口：`查看收藏题目仓库`。
- 扩展收藏题数据结构，支持保存每道题的历次练习记录 `attempts`。
- 提交审阅成功后，会把当时的原始回答、AI 审阅、完整参考回答和审阅状态写入该题历史。
- 收藏仓库支持点进题目详情，查看当时练习的原始输入、AI 审阅结果和完整参考回答。
- 支持从收藏题详情点击 `重新练习`，进入专注练习页面并清空回答框，作为同一道题的新一次练习记录。
- 点击 `采纳` AI 建议后，现在会显示提示并同步更新收藏题历史中的审阅状态。

## AI 审阅与 LLM 调用链路

- 深挖了完整参考回答质量差的原因：后端在 LLM 缺失或调用失败时会静默回退到本地模板，导致结果看起来像规则生成。
- 修改为 LLM 未配置或调用失败时直接报错，不再用默认模板冒充 AI 审阅结果。
- 审阅提交现在走 `external_llm`，不再显式走本地规则审阅。
- 保留用户提交审阅后的回答内容，不再自动清空回答框。
- 移除 `保留原样` 按钮，改成 `让 AI 重新生成`，用户可以输入“哪里不对”，并把该修改意见传给 LLM 重新生成。
- 审阅 prompt 增加 `revision_instruction`，用于让 AI 按用户反馈修订不足之处、建议和完整参考回答。

## 项目文档与出题逻辑

- 修复项目文档使用边界：生成面试题时不再把项目文档传给模型，因为面试官不应该看到项目文档。
